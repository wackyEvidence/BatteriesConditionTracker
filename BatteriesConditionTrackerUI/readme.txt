При первом запуске приложения нужно выбрать источник данных
1) PostgreSQL - удаленная база данных на сервере, выбирать в случае наличия подключения к интернету 
2) SqlServer - локальная база данных, работает только в случае наличия локального сервера
3) Текстовые файлы* - не требуют подключения к интернету, можно указать свою директорию, для этого убрать галочку "Использовать директорию по умолчанию"

*При использовании текстовых файлов нежелательно удалять что-либо из вкладки данные, если удаляемое значение используется в другом месте, лучше воспользоваться 
редактированием. Например, если у модели указан тип клемм F1, то на вкладке "Типы клемм" лучше не удалять его и добавлять новый, а изменить.

Логин и пароль для входа - admin, admin.

Каждый день приложение обновляет информацию о статусе замены АКБ: оно проверяет, когда выполнялось последнее обновление, и, если это было не сегодня, 
изменяет статус замены на "Требует замены" в том случае, если срок службы аккумулятора закончился.

После успешной авторизации откроется главная форма: для того, чтобы добавить аккумулятор, нужно нажать на кнопку "Добавить" под списком. 
При двойном нажатии на строку таблицы информация об АКБ откроется в режиме просмотра.
При добавлении нужно указать модель (добавляется во вкладке "Данные" или по нажатию на "Добавить новую"), объект, подсистему, ответственного работника 
(при отсутствии значений нужно добавить их во вкладке "Данные"). 

При добавлении модели указывается два параметра, с помощью которых приложение определяет и прогнозирует потребность в замене АКБ, поэтому желательно 
указать максимально точные значения. Это параметры группы "Срок службы": "В буферном режиме (лет)" и "Минимальное значение SoH". Если будет добавлен замер,
в котором SoH будет меньше минимального значения для модели, то аккумулятор будет считаться требующим замены. С помощью срока службы и даты начала эксплуатации
рассчитывается потребность замены при составлении отчета.

На главной форме справа от таблицы есть список фильтров, с помощью которых можно отфильтровать список аккумуляторов.

Для того, чтобы составить отчет, нужно выбрать соответствующий пункт меню, в форме создания отчета указать либо через сколько прогнозировать потребность замены,
либо на какое число, программа выбирает способ по последнему выбранному пользователю значению.

На какие моменты можно обратить внимание при презентации:
1) 3 способа хранения данных, полностью аналогичная работа приложения при выборе любого из них.
2) Приложение не хранит пароли пользователей напрямую: используется хеширование. Хеширование происходит с помощью хеш-функции bcrypt.
3) Взаимодействие с БД (добавление, обновление и удаление данных) происходит с помощью хранимых процедур. Для каждой СУБД в данный момент их 38.
4) Для маппинга объектов используется Dapper
5) Архитектура приложения соответствует принципу Open-Closed (Open-closed principle), для добавления какого-то нового источника данных достаточно реализовать один интерфейс.
